Тест-кейсы — API проект

TC-001 — Проверка GET-запроса

**Описание:**
Проверка получения данных по API.

**Метод:**
GET

**URL:**
https://jsonplaceholder.typicode.com/posts/1

**Шаги:**
1. Открыть Postman.
2. Выбрать метод GET.
3. Ввести URL.
4. Нажать Send.

**Ожидаемый результат:**
- Сервер возвращает статус 200.
- В ответе присутствуют поля userId, id, title и body.
- Значение id равно 1.

**Фактический результат:**
Все проверки выполнены успешно.

**Статус:**
Пройден.

## TC-002 — Проверка несуществующей записи

**Описание:**
Проверка ответа API при запросе записи, которой не существует.

**Метод:**
GET

**URL:**
https://jsonplaceholder.typicode.com/posts/999999

**Шаги:**
1. Открыть Postman.
2. Выбрать метод GET.
3. Ввести URL.
4. Нажать Send.

**Ожидаемый результат:**
API возвращает пустой объект или сообщает, что запись не найдена.

**Фактический результат:**
Сервер вернул статус 404 Not Found и пустой объект {}.

**Статус:**
Пройден.
